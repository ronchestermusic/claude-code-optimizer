// WORKFLOW

Seven steps. One skill. Zero surprise installs.

Stop manually checking marketplaces every week. Stop forgetting which MCP server you tried in February. Stackshift profiles how you use Claude Code, runs the whole research + install loop, and ends with a versioned PDF written for your tech level.

STEP 00
Profile
Asks your use case, tech level, and PDF style. Every later step respects the answer.
STEP 01
Audit
Scans plugins, skills, MCP servers, marketplaces — and flags what's broken.
STEP 02
Research
Parallel-searches GitHub, X, Reddit, Hacker News, pulsemcp.com, and official docs.
STEP 03
Compare
Filters by your profile, diffs against your last run, ranks by fit.
STEP 04
Ask
Numbered table. Pick by number. Nothing installs without confirmation.
STEP 05
Install
Handles plugins, MCP servers, and git-cloned skills. Flags any failures.
STEP 06
Report
Personalized PDF — change log + usage guide, language matched to your tech level.
// ROADMAP

Cross-tool compatibility is coming.

Today Stackshift runs inside Claude Code because that's where skills live. But most MCP servers already work in Cursor, Windsurf, Zed, and any MCP-compatible terminal.

Planned: detect your AI coding tools, tag every recommendation with which ones it works in, emit install instructions per tool, and print one unified PDF covering your entire AI-assistant stack. Open an issue with the tools you want supported.

// WHY

Built by an artist who got tired of tracking tools instead of making music.

I'm an artist and studio owner in Toronto building tools for the music business. Claude Code is the spine of how I work — sketching releases, automating studio admin, prototyping ideas for other artists.

But new plugins ship weekly. New MCP servers ship weekly. Tracking all of it is a part-time job nobody asked for.

So I built a skill that does the tracking for me, on a schedule I control, with receipts (the PDF) so I remember what I added and why.
